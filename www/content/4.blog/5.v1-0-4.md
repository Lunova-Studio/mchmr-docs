## 新内容
祝大家国庆节快乐，在这个国庆节我们爆更第二弹，发布了 McHMR 最新的测试版，这是本次更新的主要内容：

1. 更新器对 Token 失效后的提示优化
2. 管理员操作的 API 接入系统操作记录
3. 新增更新方式选择（多文件下载 或 增量包下载）
4. 新增多文件下载方式（此版本及以后版本默认）
5. 调整了部分 UI，优化视觉体验
6. 优化部分逻辑，提高系统稳定性
7. 修复未填写服务器名称和白名单列表导致的版本创建异常
8. 修复在 Linux 操作系统下，无法正常创建版本的问题

### 更新说明
::alert{type="info" icon="lucide:info"}
  1.0.4 公测包中已携带 SQL 更新文件（Fix 1.0.3.sql）
::

下载最新版本的 McHMR 后，打开`管理端后端`文件夹，进入`SQL`文件夹，在终端中运行SQL执行命令，更新 mchmr 数据库。
```bash
mysql -h localhost -u {数据库用户名} -p -D mchmr < "Fix 1.0.3.sql"
```
或者使用 MySQL 客户端工具，如 Navicat 等，执行 SQL 文件进行更新。

其他文件直接替换即可。

::alert{type="warning" icon="lucide:triangle-alert"}
  1. 请备份数据库后再进行更新；
  3. 若修改了启动脚本，需要根据新脚本重新配置启动参数。
::
