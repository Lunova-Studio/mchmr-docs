## 新内容
1. 新增严格模式
2. 新增版本信息显示
3. 新增限制的多文件下载时，同一个 UUID 的下载次数
4. 修复 Linux 系统下，多文件无法正常下载的的问题
5. 修复 Linux 系统下，无法正确生成增量包的的问题
6. 修复创建新版本后，有概率缓存文件有误的的问题
7. 修复部分情况下 API 生成错误的的问题
8. 修复多文件下载时，更新器下载非总进度的问题

### 更新说明
::alert{type="info" icon="lucide:info"}
  1.0.5 公测包中已携带 SQL 更新文件（Fix 1.0.4.sql）
::

下载最新版本的 McHMR 后，打开`管理端后端`文件夹，进入`SQL`文件夹，在终端中运行SQL执行命令，更新 mchmr 数据库。
```bash
mysql -h localhost -u {数据库用户名} -p -D mchmr < "Fix 1.0.4.sql"
```
或者使用 MySQL 客户端工具，如 Navicat 等，执行 SQL 文件进行更新。

其他文件直接替换即可。

::alert{type="warning" icon="lucide:triangle-alert"}
  1. 请备份数据库后再进行更新；
  3. 若修改了启动脚本，需要根据新脚本重新配置启动参数。
::
